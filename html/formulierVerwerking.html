Opmerkingen

Ik heb voor de huidige methode gekozen omdat dit het makkelijkst is om met javascript te voltooien. Aangezien javaFX niet een taal is waar we als cursist veel tijd voorkrijgen en php als taal afgeraden werd heb ik de formulier verwerking zo opgelost. Als dit om echte gebruikers info gaat mag je natuurlijk nooit een get request gebruiken maar moet je natuurlijk ook de data valideren op front en backend. en moet je het sanitizen op de backend. Dit ligt allebij een beetje buiten de scope van wat we in de theorie geleerd kregen en leek me voor deze eind opdracht niet nodig.

De huidige verwerkingspagina is een 2de pagina dit is omdat dit in praktijk vaak zo gaat deze 2de pagina is vaak een bevestigings pagina of de huidige pagina heeft een actief server script wat je doorstuurd wanneer het verzoek correct is. gezien dit in huidige vorm niet kan zonder php en javaFX is voor de huidige vorm gekozen.

Naar deze pagina wordt ook vanuit de home gelinkt dit is omdat de eindopdracht hier specifiek om vraagt dit zou je in praktijk nooit doen
